# 대용량 마크다운 청크 렌더링 (2026-03-05)

## 변경 배경
- 긴 마크다운을 한 번에 `innerHTML`로 렌더하면 메인 스레드 블로킹이 발생할 수 있습니다.
- 문서가 클수록 스크롤/입력 반응성이 떨어지는 문제가 발생합니다.

## 적용 내용
- `web/static/app.js`
  - 임계치 상수 추가
    - `LARGE_MARKDOWN_THRESHOLD` (200KB)
    - `LARGE_RENDER_CHUNK_SIZE`
  - 청크 유틸 추가
    - `splitMarkdownChunks()`
    - `renderLargeMarkdownInChunks()`
    - `nextFrame()`
  - `renderViewerMarkdown()`에서 대용량 분기 추가
    - 임계치 초과 시 TOC를 숨기고 청크 렌더링 모드로 전환
- `web/static/style.css`
  - 대용량 렌더링 안내 배너(`.viewer-large-notice`) 스타일 추가
  - 청크 블록 간격(`.viewer-chunk`) 스타일 추가

## 기대 효과
- 긴 문서 렌더링 시 UI 블로킹을 줄여 체감 반응성이 개선됩니다.
- 대용량 모드를 사용자에게 명시적으로 안내해 동작을 이해하기 쉽습니다.
