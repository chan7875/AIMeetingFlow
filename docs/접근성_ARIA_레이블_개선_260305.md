# 접근성(ARIA) 레이블 개선 개발 내역

> 개발일: 2026-03-05

---

## 개요

WCAG 접근성 가이드라인에 따라 ARIA 속성을 추가하고, 키보드 네비게이션을 지원하도록 개선했습니다.

## 변경 사항

| 파일 | 변경 내용 |
|------|----------|
| `web/static/index.html` | 랜드마크 역할(`role="banner"`, `navigation`, `main`), 모달 `role="dialog"`, `aria-modal`, `aria-expanded`, `aria-label` 추가 |
| `web/static/style.css` | `:focus-visible` 스타일 추가 (키보드 포커스 시각 피드백) |
| `web/static/app.js` | 트리 아이템 `role="treeitem"`, `tabindex`, `aria-expanded` 동적 설정, 방향키(↑↓) 네비게이션 지원 |

## 주요 기능

1. **ARIA 랜드마크**: `header`, `aside`, `main`에 적절한 role 속성 추가
2. **트리 접근성**: `role="tree"`, `role="treeitem"`, `role="group"` 구조 적용
3. **모달 접근성**: 모든 모달에 `role="dialog"`, `aria-modal="true"` 추가
4. **키보드 네비게이션**: 트리에서 방향키(↑↓)로 항목 이동, Enter/Space로 선택
5. **포커스 표시**: `:focus-visible` 스타일로 키보드 사용자에게 시각 피드백 제공
6. **aria-expanded**: 접기/펼치기 섹션의 상태를 스크린 리더에 전달
