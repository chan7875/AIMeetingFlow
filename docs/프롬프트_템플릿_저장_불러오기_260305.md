# 프롬프트 템플릿 저장/불러오기 기능 (2026-03-05)

## 변경 배경
- 사용자가 자주 쓰는 프롬프트를 매번 다시 입력해야 했습니다.
- 템플릿이 코드에 하드코딩되어 있어 사용자 맞춤 워크플로우 구성이 어려웠습니다.

## 적용 내용
- `web/static/index.html`
  - 프롬프트 영역 상단에 템플릿 UI 추가
    - `prompt-template-select` 선택 박스
    - `저장`, `삭제` 버튼
- `web/static/app.js`
  - 로컬 저장 키 추가
    - `PROMPT_TEMPLATE_STORAGE_KEY`
    - `PROMPT_TEMPLATE_LAST_KEY`
  - 템플릿 관련 함수 추가
    - `initPromptTemplates()`
    - `applyPromptTemplate()`
    - `savePromptTemplate()`
    - `deletePromptTemplate()`
  - 페이지 로드 시 템플릿 초기화 및 마지막 사용 템플릿 복원
- `web/static/style.css`
  - 템플릿 바/셀렉트 스타일 추가
  - 모바일에서 템플릿 바 줄바꿈 지원

## 기대 효과
- 프롬프트 재사용성이 높아져 반복 작업이 줄어듭니다.
