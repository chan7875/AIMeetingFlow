# 모바일 반응형 UI

> 작성일: 2026-02-27

---

## 기능 개요

768px 이하 모바일 화면에서 웹 뷰어를 최적화된 레이아웃으로 표시합니다.
데스크톱의 좌우 분할 레이아웃 대신, 사이드바를 슬라이드 드로어로 전환하고
콘텐츠 영역을 수직으로 배치합니다.

---

## 주요 변경 내용

### 1. 사이드바 슬라이드 드로어

- 모바일에서 사이드바가 화면 왼쪽 밖으로 숨겨짐 (기본 숨김)
- `☰` 토글 버튼으로 열기/닫기
- 반투명 오버레이 배경 표시, 탭 시 닫힘
- 파일 선택 후 사이드바 자동 닫힘 → 뷰어에 집중

### 2. 헤더 최적화

- 제목(`📓 Vault Viewer`) 숨김으로 버튼 공간 확보
- 버튼 텍스트 레이블 숨김(`btn-text`), 아이콘만 표시
- 7개 버튼(☰ ⚙ ⬆ 🛰 ⬇ ⬆)이 한 줄에 배치
- `renderAutoWatchButton()`을 `innerHTML` 방식으로 수정해 span 구조 보존

### 3. 화면 비율 (뷰포트 기준)

| 영역 | 높이 |
|------|------|
| 헤더 | 52px (고정) |
| 마크다운 뷰어 | 30vh |
| 프롬프트 입력 | 20vh |
| AI 실행 결과 | 30vh |

- AI 패널 본문이 가로 → 세로 스택으로 전환
- 마크다운 테이블 가로 스크롤 처리

### 4. 모달 개선

- 전체 너비(`width: 96%`)로 확장
- `min-width` 제거로 좁은 화면 대응

---

## 사용법

| 동작 | 결과 |
|------|------|
| `☰` 버튼 탭 | 파일 탐색기 열기 |
| 파일 목록 배경 탭 | 탐색기 닫기 |
| 파일 탭 | 탐색기 자동 닫힘 + 뷰어 표시 |
| 데스크톱 (769px 이상) | 기존 좌우 분할 레이아웃 유지 |

---

## 관련 파일

| 파일 | 변경 내용 |
|------|-----------|
| `web/static/style.css` | `@media (max-width: 768px)` 모바일 스타일 추가 |
| `web/static/index.html` | `☰` 토글 버튼, 사이드바 오버레이, `btn-text` span 추가 |
| `web/static/app.js` | `toggleSidebar()`, `closeSidebar()`, `isMobile()` 함수 추가, 파일 열기 시 자동 닫힘, `renderAutoWatchButton()` innerHTML 수정 |
