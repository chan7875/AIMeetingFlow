# AI 결과 UX 마무리: 복사/저장 동작 정리 (2026-03-05)

## 변경 배경
- AI 결과를 바로 복사하려면 수동 선택이 필요했습니다.
- `issue 저장` 버튼 동작이 AI 실행 결과 저장이 아니라 재요약 실행이어서 사용자가 기대한 흐름과 달랐습니다.

## 적용 내용
- `web/static/index.html`
  - 실행결과 섹션에 `AI 결과 복사` 버튼(`copy-result-btn`) 추가
- `web/static/app.js`
  - 결과 액션 버튼 상태 함수 추가: `updateResultActionButtons()`
  - 복사 기능 추가: `copyAIResult()`
  - `saveToIssue()` 동작 정리
    - `state.aiResult`가 있으면 `/api/ai/save-result`로 현재 결과를 즉시 저장
    - 결과가 없을 때만 기존 `/api/ai/summarize` 경로 사용
  - AI 실행 시작/초기화/렌더 시 버튼 표시 상태 동기화
- `web/routers/ai.py`
  - `POST /api/ai/save-result` 추가
    - 입력: `file_path`, `ai_output`
    - 빈 결과 방어/경로 검증/이슈 파일 저장 처리

## 기대 효과
- AI 결과 복사가 즉시 가능해져 후속 공유/전달 속도가 빨라집니다.
- 사용자가 본 결과를 그대로 저장하는 흐름이 명확해져 UX 혼란이 줄어듭니다.
