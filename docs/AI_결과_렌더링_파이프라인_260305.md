# AI 결과 렌더링 파이프라인 정리 (2026-03-05)

## 변경 배경
- AI 실행 결과가 화면에 표시되는 경로가 `runAI`, `runAIStream`으로 나뉘어 있었고,
- 결과 마크다운 렌더링/원문 보관 로직이 중복되어 유지보수 난이도가 높았습니다.

## 적용 내용
- `web/static/app.js`에 `renderAIResult(rawText)` 공통 함수를 추가했습니다.
  - `state.aiResult`에 원문 텍스트를 저장
  - `marked.parse()`로 마크다운 렌더링
  - 코드블록 `highlight.js` 재적용
- `runAI()`는 내부적으로 `runAIStream()`을 호출하도록 단일 실행 경로로 정리했습니다.
- `runAIStream()`의 `done` 이벤트는 `renderAIResult()`를 사용하도록 변경했습니다.

## 기대 효과
- AI 결과 표시 경로가 단일화되어 동작 일관성이 개선됩니다.
- 향후 렌더링 정책 변경 시 한 함수만 수정하면 되어 유지보수가 쉬워집니다.
