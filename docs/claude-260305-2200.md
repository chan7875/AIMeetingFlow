## 1. 높은 우선순위 (핵심 기능 확장)

### 1.1 플랫폼 배포 서비스 구현

CLAUDE.md에 명시된 멀티 플랫폼 배포 서비스가 아직 미구현 상태입니다.

| 서비스 | 구현 내용 | 기술 스택 |
|--------|-----------|-----------|
| `youtube_service.py` | YouTube 업로드, 메타데이터 관리 | OAuth2, yt-dlp, ffmpeg |
| `threads_service.py` | Threads 게시, 450자 분할 | Meta Graph API |
| `naver_service.py` | 네이버 블로그 자동 포스팅 | Playwright 브라우저 자동화 |
| `linkedin_service.py` | LinkedIn 게시, 쿨다운 관리 | OAuth2 (1~72시간 쿨다운) |
| `scheduler_service.py` | 전체 배포 스케줄링 | APScheduler |

### 1.2 설정 모듈 생성

- `config/settings.py` — 환경변수 기반 중앙 설정 관리 (서비스 계층에서 import)
- `models/prompt_settings.py` — 프롬프트 템플릿 모델 정의 (chatgpt_service에서 사용)

### 1.3 인증/인가 시스템

현재 누구나 접근 가능한 상태이므로:
- 기본 인증 레이어 추가 (JWT 또는 세션 기반)
- API 키 관리 암호화 (현재 `/data/`에 평문 저장)
- CSRF 토큰 검증

## 2. 중간 우선순위 (품질 및 성능)

### 2.1 테스트 강화

현재 22개 테스트 파일이 있으나 대부분 수락 테스트 수준입니다.

- **서비스 단위 테스트**: Claude CLI, Codex CLI, ChatGPT 서비스 각각의 단위 테스트
- **라우터 테스트**: 에러 핸들링, 경로 검증, 동시성 처리
- **보안 테스트**: 경로 탐색(path traversal) 공격, 입력 길이 제한
- **통합 테스트**: 자동감시 + 슬라이드 생성 파이프라인 E2E 테스트

### 2.2 성능 최적화

| 대상 | 현재 문제 | 개선 방안 |
|------|-----------|-----------|
| 자동감시 | 2초 폴링 (비효율적) | FSEvents/inotify 기반 감시로 전환 |
| 검색 | `os.walk()` 매번 호출, O(n) | 인덱싱 및 결과 캐싱 도입 |
| 마크다운 렌더링 | 클라이언트 전용, 대용량 시 블로킹 | 서버사이드 캐싱, Web Worker 활용 |
| 서브프로세스 | 전체 버퍼 수집 후 응답 | 스트리밍 레이어 개선 |

### 2.3 에러 처리 고도화

- **서킷 브레이커 패턴**: 외부 서비스(OpenAI, Claude CLI) 반복 실패 시 일시 차단
- **지수 백오프**: NLM 재시도 시 고정 간격 대신 지수 백오프 적용
- **구조화된 로깅**: stdout 출력 대신 파일 기반 로깅 (loguru 또는 structlog)
- **알림 연동**: 자동감시 에러 시 Slack/이메일 알림
