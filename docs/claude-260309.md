## 2. UI/UX 개선 분석

### 2.1 높은 우선순위 개선사항

#### 2.1.1 키보드 단축키 시스템

**현재 문제:** Ctrl+K (파일 검색)만 존재. 핵심 기능에 대한 키보드 접근성 부재.

**개선안:**
| 단축키 | 기능 |
|--------|------|
| `Ctrl+Enter` | AI 실행 |
| `Ctrl+S` | 파일 저장 / Issue 저장 |
| `Ctrl+Shift+S` | AI 결과 Issue 저장 |
| `Ctrl+B` | 사이드바 토글 |
| `Ctrl+J` | AI 패널 토글 |
| `Ctrl+Shift+F` | 볼트 내용 검색 |
| `Esc` | 모달 닫기 / AI 중단 |

#### 2.1.2 로딩 상태 UX 개선

**현재 문제:** AI 실행 중 "실행 중..." 텍스트만 표시. 진행 상황 불투명.

**개선안:**
- AI 스트리밍 중 경과 시간 타이머 표시
- 토큰/글자 수 실시간 카운터
- 프로그레스 인디케이터 (스피너 → 스켈레톤 UI)
- 슬라이드 생성 시 단계별 진행 표시 (소스 추가 → 생성 중 → 완료)

#### 2.1.3 파일 트리 UX

**현재 문제:** 폴더 접기/펼치기 상태가 새로고침 시 초기화됨. 대규모 볼트에서 탐색 비효율.

**개선안:**
- 폴더 접기/펼치기 상태 localStorage 저장
- 파일 아이콘 (확장자별 구분: .md, .txt, .pdf, .pptx, 이미지)
- 드래그 앤 드롭 파일 이동
- 현재 열린 파일 위치로 자동 스크롤 + 하이라이트

#### 2.1.4 AI 패널 개선

**현재 문제:** 프롬프트 입력과 결과가 수직 레이아웃으로 고정. 화면 공간 비효율.

**개선안:**
- 프롬프트/결과 영역 리사이즈 가능 (드래그 핸들)
- AI 결과 마크다운 렌더링 토글 (Raw ↔ Rendered)
- 결과 내 코드 블록 원클릭 복사
- 프롬프트 히스토리 (최근 10개 자동 저장)

### 2.2 중간 우선순위 개선사항

#### 2.2.1 에디터 개선

**현재 문제:** 단순 `<textarea>` 기반 편집. 마크다운 미리보기 없음.

**개선안:**
- 마크다운 실시간 미리보기 (Split editor)
- 기본 마크다운 단축키 (Bold, Italic, Link, Code)
- 줄번호 표시
- 자동 저장 (debounce 2초)

#### 2.2.2 알림 센터

**현재 문제:** Toast 알림이 일회성으로 사라짐. 자동감시/슬라이드 상태 확인 어려움.

**개선안:**
- 헤더에 알림 벨 아이콘 + 뱃지 카운트
- 알림 드롭다운 (최근 20건 히스토리)
- 카테고리 필터 (AI, Git, AutoWatch, Slides)
- 알림 클릭 시 해당 파일/결과로 이동

#### 2.2.3 브레드크럼 네비게이션

**현재 문제:** 현재 파일 경로가 뷰어 헤더에 작은 텍스트로만 표시.

**개선안:**
- 클릭 가능한 브레드크럼 경로 (폴더 클릭 시 해당 폴더 열기)
- 경로 복사 버튼

#### 2.2.4 다중 탭 뷰어

**현재 문제:** 한 번에 하나의 파일만 열 수 있음.

**개선안:**
- 탭 형태로 여러 파일 동시 열기 (최대 10개)
- 탭 닫기, 탭 순서 변경
- 수정 중인 파일 탭에 인디케이터 표시

### 2.3 낮은 우선순위 개선사항

| 항목 | 설명 |
|------|------|
| 즐겨찾기/핀 기능 | 자주 사용하는 파일을 사이드바 상단에 고정 |
| 파일 비교 도구 | 두 파일 간 diff 비교 뷰 |
| 태그 필터 | YAML frontmatter의 tags 기반 파일 필터링 |
| 워드 카운트 | 현재 파일의 글자/단어/라인 수 상태바 표시 |
| 인쇄 뷰 | 마크다운 파일 인쇄 최적화 CSS |
| 전체화면 모드 | F11 전체화면 뷰어 (집중 모드) |

## 3. 추가 기능 추천

### 3.1 높은 우선순위

#### 3.1.1 CLI 도구 (구현 완료)

**파일:** `cli.py`

터미널에서 프로젝트의 모든 핵심 기능을 실행할 수 있는 통합 CLI 도구.

```
aimf config show                          # 설정 조회
aimf config set --key vault_path --value ~/vault  # 설정 변경
aimf tree                                 # 파일 트리 출력
aimf read <path>                          # 파일 읽기
aimf search <query>                       # 텍스트 검색
aimf ai --file <path> --engine claude     # AI 분석 (실시간 스트리밍)
aimf summarize <path> --slides            # AI 요약 + Issue 저장 + 슬라이드
aimf git status|pull|push|changes         # Git 작업
aimf slides generate --file <path>        # 슬라이드 생성
aimf serve --port 8101                    # 웹 서버 시작
```

**구현 내용:**
- 9개 서브커맨드: `config`, `tree`, `read`, `search`, `ai`, `summarize`, `git`, `slides`, `serve`
- AI 실시간 스트리밍 출력 (subprocess stdout 라인별 출력)
- 볼트 경로 보안 검증 (path traversal 방지)
- stdin 파이프 지원 (`cat file.md | aimf ai`)
- 기존 서비스 레이어 재사용 (중복 코드 최소화)

#### 3.1.2 배치 처리 기능

**현재 한계:** 파일 하나씩만 AI 분석 가능.

**추천:**
- 폴더 전체 일괄 AI 요약 (`aimf summarize --dir <folder>`)
- 병렬 처리 옵션 (`--parallel 3`)
- 진행 상황 프로그레스바
- 결과 리포트 (성공/실패 건수, 소요 시간)

#### 3.1.3 AI 결과 히스토리

**추천:**
- AI 실행 결과를 타임스탬프와 함께 자동 저장
- 이전 결과 조회/비교 (web UI + CLI)
- 프롬프트별 결과 통계
- 히스토리 검색

### 3.2 중간 우선순위

## 4. CLI 도구 상세 사양

### 4.1 아키텍처

```
cli.py (엔트리포인트)
├── argparse 기반 서브커맨드 파서
├── web/config.py 재사용 (설정 관리)
├── web/routers/ai.py 재사용 (summarize_file_to_issue)
├── services/notebooklm_service.py 재사용 (슬라이드 생성)
└── subprocess 직접 호출 (AI 스트리밍, Git)
```

### 4.2 명령어 상세

| 명령어 | 설명 | 주요 옵션 |
|--------|------|-----------|
| `config show` | 전체 설정 JSON 출력 | — |
| `config set` | 개별 설정 변경 | `--key`, `--value` |
| `tree` | 볼트 파일 트리 (텍스트) | — |
| `read <path>` | 파일 내용 stdout 출력 | — |
| `search <query>` | 볼트 텍스트 검색 | `--limit` |
| `ai` | AI 분석 (실시간 스트리밍) | `--engine`, `--file`, `--text`, `--prompt` |
| `summarize <file>` | AI 요약 + Issue 저장 | `--engine`, `--prompt`, `--timeout`, `--slides` |
| `git <action>` | Git 작업 | `--message`, `--files` |
| `slides <action>` | 슬라이드 관리 | `--file`, `--account` |
| `serve` | 웹 서버 시작 | `--port`, `--vault`, `--reload` |

### 4.3 입력 방식

1. **파일 지정:** `--file <볼트 내 상대 경로>`
2. **텍스트 직접 입력:** `--text "분석할 내용"`
3. **파이프 입력:** `echo "내용" | aimf ai`
4. **대화형:** stdin이 tty일 때 에러 메시지로 안내

### 4.4 보안

- 모든 경로 접근 시 볼트 루트 기준 path traversal 검증
- `CLAUDECODE` 환경변수 제거로 nested session 방지
- Git push 전 대상 파일 목록 출력
